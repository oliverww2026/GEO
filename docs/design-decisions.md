# GEO 项目设计决策文档

## 1. 品牌信息配置的简化

### 问题背景
最初的设计中包含了三个品牌信息输入字段：
- 品牌名称
- 品牌定位（高价回收/上门服务等）
- 服务城市

### 决策：移除手动输入，改为权限系统自动识别

#### 原因分析

1. **品牌名称**
   - ✅ **必需且有用**：在评分维度4（品牌锚点强度）中使用，用于检测品牌名在内容中的出现情况
   - ❌ **不应手动输入**：实际使用场景中，用户（如久久金的员工ABCD）每次使用时都是为自己公司做检测，不需要重复输入品牌名
   - ✅ **解决方案**：通过权限系统自动识别用户所属品牌，作为后端 API 的入参

2. **品牌定位和服务城市**
   - ❌ **在评分逻辑中未使用**：查看 `prompt/geo-commercial-prompt-v2.3.md` 发现，这两个字段只在初始化时收集，但在实际的 6 个评分维度中都没有被使用
   - ❌ **增加用户负担**：要求用户每次填写这些信息，但实际上对分析结果没有影响
   - ✅ **解决方案**：完全移除这两个字段

### 实施方案

#### 前端界面
- 移除"品牌信息配置"表单区域
- 在页面顶部显示当前品牌信息（通过权限系统自动识别）
- 简化为两步流程：
  1. 内容输入
  2. 投放渠道选择

#### 后端 API 设计
```typescript
// API 请求
POST /api/analyze
Headers: {
  Authorization: "Bearer <token>"  // 包含用户身份信息
}
Body: {
  content: string,      // 要分析的内容
  channels: string[]    // 投放渠道
}

// 后端处理
1. 从 token 中解析用户身份
2. 查询用户所属品牌（如：久久金）
3. 使用品牌名称进行内容分析
4. 返回分析结果
```

### 用户体验改进

**优化前：**
```
【步骤1】品牌信息配置
  - 品牌名称: [输入框]
  - 品牌定位: [下拉选择]
  - 服务城市: [输入框]

【步骤2】内容输入
  - [大文本框]

【步骤3】投放渠道
  - [多选框]
```

**优化后：**
```
ℹ️ 当前品牌：久久金（通过权限系统自动识别）

【步骤1】内容输入
  - [大文本框]

【步骤2】投放渠道
  - [多选框]
```

### 优势

1. **减少用户操作**：从 6 个输入项减少到 2 个
2. **避免重复输入**：用户不需要每次都输入品牌名称
3. **提高准确性**：避免用户输入错误的品牌名称
4. **符合实际使用场景**：企业员工使用时，品牌信息是固定的
5. **移除无用字段**：品牌定位和服务城市在评分逻辑中未使用

## 2. 评分维度说明

根据 `prompt/geo-commercial-prompt-v2.3.md`，实际使用的评分维度为：

1. **信源权威性（30%）** - 基于投放渠道判断
2. **内容类型匹配度（25%）** - 基于内容结构分析
3. **数据丰富度（15%）** - 基于内容中的数据数量和来源
4. **品牌锚点强度（10%）** - 基于品牌名称在内容中的出现情况 ✅ **使用品牌名称**
5. **篇幅匹配度（10%）** - 基于内容字数
6. **营销语气风险（10%）** - 基于营销词库检测

只有维度4需要品牌名称，其他维度都不需要品牌定位和服务城市信息。

## 4. 技术实现要点

### 前端
```typescript
// HomePage.tsx
const brandName = '久久金' // TODO: 从权限系统获取

// 提交分析时
const response = await api.analyze({
  content: values.content,
  channels: values.channels
  // brandName 不需要传递，后端从 token 中获取
})
```

### 后端（伪代码）
```python
@app.post("/api/analyze")
async def analyze_content(
    request: AnalyzeRequest,
    current_user: User = Depends(get_current_user)
):
    # 从用户信息中获取品牌
    brand_name = current_user.brand.name
    
    # 执行分析
    result = analyzer.analyze(
        content=request.content,
        channels=request.channels,
        brand_name=brand_name  # 使用权限系统中的品牌名
    )
    
    return result
```
