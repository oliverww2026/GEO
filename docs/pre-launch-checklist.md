## 🚀 一键部署上线

### 方式1：完整部署（推荐首次使用）

```bash
# 在项目根目录执行
./deploy.sh
```

**这个脚本会自动**：
1. 安装后端依赖
2. 构建后端TypeScript代码
3. 安装前端依赖
4. 构建前端生产版本
5. 创建日志目录

**预计耗时**：2-5分钟（取决于网络速度）

---

### 方式2：启动服务

```bash
# 部署完成后，启动生产服务
./start-production.sh
```

**这个脚本会自动**：
1. 启动后端服务（端口3001）
2. 启动前端服务（端口4173）
3. 后台运行，日志输出到logs目录

**预计耗时**：10-20秒

---

### 方式3：停止服务

```bash
# 需要停止时执行
./stop-production.sh
```

## 🐛 常见问题排查

### 问题1：端口被占用

**现象**：启动失败，提示端口已被使用

**解决**：
```bash
# 查找占用端口的进程
lsof -i :3001  # 后端
lsof -i :4173  # 前端

# 杀死进程
kill -9 <PID>

# 重新启动
./start-production.sh
```

---

### 问题2：依赖安装失败

**现象**：`npm install` 报错

**解决**：
```bash
# 清理缓存
npm cache clean --force

# 删除node_modules重新安装
rm -rf backend/node_modules frontend/node_modules
./deploy.sh
```

---

### 问题3：前端无法连接后端

**现象**：前端显示"网络错误"

**检查**：
```bash
# 1. 确认后端是否启动
curl http://localhost:3001/health

# 2. 检查前端API配置
cat frontend/src/App.tsx | grep "localhost:3001"

# 3. 查看后端日志
tail -f logs/backend-*.log
```

---

### 问题4：营销词检测不准确

**现象**：明显的营销词没有被检测到

**检查**：
```bash
# 查看词库内容
cat data/marketing_wordlib.json | jq '.strong' | head -20

# 确认词库路径正确
ls -la data/marketing_wordlib.json
```
