🔀 路由架构

App (AuthProvider → BrowserRouter) │ ├── 无 Header 布局（全屏页面） │ ├── / → LandingPage │ ├── /login → LoginPage │ └── /register → RegisterPage │ └── 带 Header 布局 ├── /app → ProtectedRoute → HomePage ├── /result → ProtectedRoute → ResultPage └── /admin → ProtectedRoute(admin) → AdminPage
设计要点：NO_HEADER_PATHS 数组控制布局切换；ProtectedRoute 组件包裹需认证页面，支持 requireAdmin 角色校验

🔐 认证与授权

AuthContext 全局状态

  • token — JWT Token
  • user — 用户信息 (userId, role)
  • enterprise — 企业信息 (brandName, serviceCity)
  • isAuthenticated — 登录态

核心方法

  • login(username, password)
  • register(inviteCode, ...)
  • logout() — 清空 localStorage
  • getAuthHeaders() — Bearer Token
持久化：token/user/enterprise 存于 localStorage，启动恢复 + 后台 token 验证。
路由守卫：ProtectedRoute 检查 isAuthenticated，未登录 → /login
